### 🎼 New Track: Garbage Collection Symphony

#### Concept
**"Garbage Collection Symphony"** is a musical representation of the memory management process that runs continuously in computer systems. The track transforms the technical process of memory allocation, usage, and cleanup into a beautiful symphony.

#### Musical Elements
- **Memory Allocation**: New melodic phrases appear as memory blocks are allocated
- **Memory Usage**: Overall texture density represents memory consumption
- **Garbage Collection**: Sweeping, cleaning patterns when the collector runs
- **Memory Freeing**: Releases create resonant spaces in the music
- **Fragmentation**: Dissonance when memory is scattered and inefficient

#### Technical Implementation

##### MemoryBlock Class
Each memory block is represented with:
- **Lifetime**: Duration based on block size
- **Frequency Content**: Multiple harmonics based on memory characteristics
- **State Management**: Tracks allocation, usage, and collection status
- **Fragmentation**: Some blocks start fragmented, creating dissonance

##### GarbageCollector Class
Implements different collection algorithms with unique sounds:
1. **Sweep Pattern**: Classic mark-and-sweep with frequency sweeps
2. **Mark and Sweep**: Staccato "mark" notes followed by "sweep" sounds
3. **Generational Pattern**: Different generations have different frequency ranges
4. **Concurrent Pattern**: Multiple collection threads running simultaneously

##### Symphony Structure
- **Build Phase** (0-10s): Gradual increase in memory allocation and musical density
- **Climax Phase** (15-25s): Peak memory usage with complex interactions
- **Resolution Phase** (27-32s): Final cleanup and graceful fade-out

#### Audio Characteristics
- **Duration**: 32 seconds
- **Complexity**: 94 memory blocks allocated throughout the piece
- **Collection Events**: 8 collection cycles with varying algorithms
- **Texture**: Evolves from sparse to dense to resolved
